## 8.1.**热更新系统资产（Hot Update Assets）**

当下手游的迭代高度依赖热更新， **前期不做，后期版本节奏全靠人肉拯救** 。一个完善的热更新系统，直接决定了项目的迭代效率、版本稳定性和运营响应能力。

---

### **热更新系统构成**

热更新系统是一个“跨端 + 跨流程 + 跨职能”的工程系统，主要包含以下4个组成部分：

| **模块** | **内容**                         | **常用方案**             | **技术要点**                     |
| -------------- | -------------------------------------- | ------------------------------ | -------------------------------------- |
| 代码热更       | 客户端逻辑层（C# 或 Lua）              | ILRuntime、XLua、HybridCLR     | 插件接入、虚拟机运行、代码剥离与加载   |
| 配表热更       | Excel 表格 → JSON/Lua                 | 自研构建脚本/ 表校验工具       | 策划主导、字段校验、结构标准化         |
| 资源热更       | UI、Prefab、模型、音效等               | Unity Addressable、AssetBundle | 资源依赖构建、分包策略、CDN管理        |
| 热更管理后台   | 版本控制、灰度发布、差分更新、回滚机制 | 自研/ TapTap构建平台 / 腾讯TPM | 后台面板、资源校验、权限控制、数据追踪 |

---

### **推荐技术选型与资源获取**

#### **代码热更**

 **推荐方案** ：HybridCLR（推荐），或 ILRuntime、XLua（成熟）

 **说明** ：HybridCLR 支持 AOT+Interpreter 模式，兼容性高，性能接近原生。

 **获取** ：[https://github.com/focus-creative-games/hybridclr](https://github.com/focus-creative-games/hybridclr)

#### **配表热更**

 **推荐工具** ：Excel2Json、table-converter（开源）、飞书转JSON脚本（团队自研）

 **说明** ：建议设计通用表头、数据校验工具（如字段类型、主键重复）

 **示例脚本** ：[https://github.com/egametang/Excel2Json](https://github.com/egametang/Excel2Json)

#### **资源热更**

 **推荐方案** ：Unity Addressable + 分包结构设计 + CDN托管

 **说明** ：使用标签（Label）标记热更资源，构建 remote load group

 **官方指南** ：[https://docs.unity3d.com/Manual/com.unity.addressables.html](https://docs.unity3d.com/Manual/com.unity.addressables.html)

#### **热更平台（后台管理）**

 **推荐路径** ：

MVP阶段使用 Git + 文件校验方式

商业化后可接入 TapTap构建平台 或 自研后台

 **TapTap平台** ：[https://www.taptap.cn/developer](https://www.taptap.cn/developer)

 **自研后台功能建议** ：版本灰度、强制更新、回滚、日志上报、资源校验

---

### **成本与开发建议**

| **模块** | **成本级别**      | **推荐落地时机**     |
| -------------- | ----------------------- | -------------------------- |
| 代码热更       | ☆☆☆☆（技术重度）    | 初版上线前（基础逻辑剥离） |
| 配表热更       | ☆☆（低）              | 立项初期即可落地           |
| 资源热更       | ☆☆☆☆（需设计配合）  | 美术产线上线前落地         |
| 热更后台       | ☆☆☆☆☆（后端+运维） | 首发后视体量逐步接入       |

### **总结**

一个完整的热更新系统，不只是技术选型，更是一种 **组织效率的放大器** 。越早投入、越早系统化，就越容易支撑高频迭代、跨职能合作与内容长线运营。

建议从表格热更 + 资源热更最先起步，再逐步引入代码热更和版本控制后台。
